# Иконки

## Структура

```
icons/
├── icons.js     ← скрипт-агрегатор: собирает все .svg в объект { 'размер/имя': компонент }
├── 24/          ← иконки 24×24
├── 32/          ← иконки 32×32
├── 40/          ← иконки 40×40
└── ...          ← добавляй папки под нужные размеры
```

## Как добавить иконку

1. Положи SVG-файл в папку нужного размера: `app/assets/icons/24/arrow-right.svg`.
2. Готово. Скрипт `icons.js` подхватит файл автоматически (через `import.meta.glob`).
3. Используй в шаблоне:

```vue
<Icon name="24/arrow-right" />
```

Имя — это путь относительно `icons/` без расширения: `24/arrow-right`, `32/close`, `40/burger`.

## Подготовка SVG

- Убедись, что в SVG нет лишних `id`, `width`/`height` (или они равны размеру папки).
- Для одноцветных иконок поставь `fill="currentColor"` — тогда иконка будет принимать `color` от родителя.
- Чистка SVG: https://jakearchibald.github.io/svgomg/ (включи «Remove width/height»).

## Размер бандла

Сейчас все иконки грузятся **eager** — попадают в основной бандл. Это удобно (одна `<Icon name="..." />` без асинхронности), но при росте числа иконок до сотен — стоит подумать об одном из вариантов:

1. **Lazy load** — заменить в `icons.js` `eager: true` на `eager: false` и сделать `<Icon>` асинхронным через `defineAsyncComponent`.
2. **SVG sprite** — собирать все иконки в один `sprite.svg` и использовать через `<use href="...">`.

Для бойлерплейта оставлено как есть.

## Альтернатива: явный импорт SVG

Иногда нужна одна иконка с особыми параметрами. В `nuxt.config.ts` подключён `vite-svg-loader`, можно так:

```vue
<script setup>
  import IconLogo from '~/assets/icons/logo.svg?component';
</script>

<template>
  <IconLogo class="logo" />
</template>
```

`?component` обязателен — без него Vite вернёт строку (URL).
